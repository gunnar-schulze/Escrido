/*# @_page_ config_file Configuration File Options

The @ref escrido_file "Escrido configuration file" is one of two ways to define settings for the execution of Escrido. The other way is to use arguments during the <em>command line call</em>. Each configuration option can be set in either way. For options that are present in the configuration file <em>and</em> in the command line call, the command line argument overrules the setting from the configuration file.

The following configuration options are available:

@table
  <b>Configuration file option</b>       | <b>Command line arguments</b> @lb
  <em>General Options</em> @lb
  @ref config_option_templatedir         | -t, --template @lb
  @ref config_option_include             | -i, --include @lb
  @ref config_option_namespace           | -ns, --namespace @lb
  @ref config_option_excludegroups       | -xg, --exclude-groups @lb
  @ref config_option_relabel             | -rl, --relabel @lb
  <em>Web Document Output Options</em> @lb
  @ref config_option_generatewebdoc      | -wd, --webdoc @lb
  @ref config_option_webdocoutdir        | -wdo, --webdoc-output-dir @lb
  @ref config_option_webdocfileending    | -wdfe, --webdoc-file-ending @lb
  @ref config_option_generatesearchindex | -si, --search-index @lb
  @ref config_option_searchindexencoding | -sie, --search-index-encoding @lb
  @ref config_option_searchindexfile     | -sif, --search-index-file @lb
  <em>LaTeX Document Output Options @lb
  @ref config_option_generatelatex       | -ld, --latex @lb
  @ref config_option_latexoutdir         | -ldo, --latex-output-dir
@endtable

@section config_options_general General Options

@subsection config_option_templatedir TEMPLATE_DIR

This parameter should point to a template directory where LaTeX document and HTML web document template files are stored.

@example
TEMPLATE_DIR = "./templates/manual/"
@par

@subsection config_option_include INCLUDE

This parameter defines a namespace the documentation shall be restricted to. If empty, all available namespaces are included. Multiple values can be given seperated by blank spaces.

@example
INCLUDE = "./examples/manual/*.txt"
@par

@subsection config_option_namespace NAMESPACE

This parameter defines a namespace the documentation shall be restricted to. This will only include pages with the respective @ref tag_namespace "namespace tag". If empty, all available namespaces are included. Multiple values can be given seperated by blank spaces.

@example
NAMESPACE = algorithms math
@par

@subsection config_option_excludegroups EXCLUDE_GROUPS

This parameter defines a list of @ref tag_ingroup "groups" that shall NOT appear in the documentation, i.e. it creates a versions of the documentation with these groups filtered out, e.g. for a group 'Internal'. If empty, all groups are included. Multiple values can be given seperated by blank spaces.

@example
GENERATE_WEBDOC = Internal
@parameter

@subsection config_option_relabel RELABEL

This parameter can be used to replace certain fixed terms or expressions that are used by default in the output documents by a different wording. For that, a text pair is given with the first text representing the original expression and the second one the replacement string. This parameter can be used multiple times.

The following original expressions can be relabeled by this option:

@table
  "Author"   | "Attributes" | "Contents" | "Copyright" @lb
  "Date"     | "Details"    | "Example"  | "Note" @lb
  "Output"   | "Parameters" | "Remark"   | "Return value" @lb
  "See also" | "Signatures" | "Version"
@endtable

@example
RELABEL = "Return value" "Returns"
@par

@section config_options_webdoc Web Document Output Options

These options control the output of the HTML web document generated by Escrido.

@subsection config_option_generatewebdoc GENERATE_WEBDOC

Flag defining whether a web document shall be created. If this flag is set to YES, Escrido will generate HTML output.

@example
GENERATE_WEBDOC = YES
@par

@subsection config_option_webdocoutdir WEBDOC_OUT_DIR

This parameter should point to a directory that will be filled with the web document output files. Default is "./html/".

@example
WEBDOC_OUT_DIR = "./manual/html/"
@par

@subsection config_option_webdocfileending WEBDOC_FILE_ENDING

This parameter specifies an alternative file ending to the web document output files, e.g. ".php". Default is ".html".

@example
WEBDOC_FILE_ENDING = ".php"
@par

@subsection config_option_generatesearchindex GENERATE_SEARCH_INDEX

This parameter turns on the generation of a search index list that can be used to implement a search functionality for the pages, e.g. in combination with some external search library. Default is NO.

@example
GENERATE_SEARCH_INDEX = YES
@par

@subsection config_option_searchindexencoding SEARCH_INDEX_ENDCODING

This parameter defines the encoding of the search index file. If this is 'JSON', the search index file contains a pure JSON array, if it is 'JS', the search index file contains a JavaScript object 'searchIndex'. Default is "JSON".

@example
SEARCH_INDEX_ENDCODING = JS
@par

@subsection config_option_searchindexfile SEARCH_INDEX_FILE

This parameter defines the file path of the search index file (relative to the HTML folder). The content of the file is a JSON search index list. Default is "srchidx.json".

@example
SEARCH_INDEX_FILE  = "js/srchidx.js"
@par

@section config_options_latex LaTeX Document Output Options

@subsection config_option_generatelatex GENERATE_LATEX

Flag defining whether a LaTeX document shall be created. If this flag is set to YES, Escrido will generate LaTeX output.

@example
GENERATE_LATEX = NO
@par

@subsection config_option_latexoutdir LATEX_OUT_DIR

This parameter should point to a directory that will be filled with the LaTeX output files. Default is "./latex/".

@example
LATEX_OUT_DIR = "./manual/latex/"
@par

#*/
